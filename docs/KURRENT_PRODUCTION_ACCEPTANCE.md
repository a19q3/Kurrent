# Kurrent Production Acceptance

Status: blocked on independent external security review

This document defines the acceptance checklist for a future production claim.
It does not state that the current repository is production-ready.

## Scope

Production acceptance requires the research specification, implementation,
evidence, operational controls, and independent review to agree on the same
source revision and target network profile. Local-devnet evidence is necessary
but not sufficient.

## Required Gates

- `cargo test` passes.
- `./scripts/check.sh` regenerates local acceptance evidence for the intended
  source revision.
- `./scripts/verify-evidence.sh` accepts only evidence bound to the current
  source revision.
- `./scripts/write-production-target-profile.sh` records the target profile.
- `./scripts/run-semantic-transaction-verifier.sh` validates raw transaction,
  witness, script, txid, and receipt relationships.
- `./scripts/run-adversarial-soak.sh` completes the deterministic adversarial
  model suite.
- Operational runbooks for key management, monitoring, recovery, and rollout
  are present.
- Independent external security review evidence passes.

## Rejection Conditions

A production claim must be rejected if any evidence is stale, any reviewed
artefact hash differs from the security review package, any critical/high/
medium/low security finding remains open, or any source change lands after
review without regenerating evidence and repeating review.

## Evidence

The authoritative production evidence is
`evidence/kurrent-production-readiness.json`, generated by
`./scripts/verify-production-readiness.sh`, plus the external review artefacts
referenced by `evidence/production/security-review.json`.
